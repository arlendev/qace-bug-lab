# Test Cases

This directory contains the manual test cases created during the development of QAce Bug Lab.

The objective of this documentation is to demonstrate a structured Quality Assurance process, including test design, feature validation and regression testing activities.

## Current Coverage

### Authentication

* Login
* Logout

### Search and Filtering

* Search Functionality
* Severity Filters

### CRUD Operations

* Create Bug
* Update Bug
* Delete Bug

### Data Persistence

* localStorage Persistence
* Seed Data Loading

## Test Case Structure

Each test case contains:

* Test Case ID
* Preconditions
* Test Steps
* Expected Result
* Execution Status

Some executed test cases may also include:

* Actual Result
* Pass / Fail Status
